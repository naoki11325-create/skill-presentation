# 出力品質チェックリスト

商談台本を出力する前に、以下を満たすか確認する。満たさない項目があれば直してから出力する。

## 結論と構造

- [ ] 冒頭に【お願い（結論）】があり、「だから相手に何をしてほしいか」まで言い切っている（要約で終わっていない）
- [ ] 業種が確定していて、その業種の得・不安・送客の中身に合っている（塾／歯科医／小児科医／市役所で中身が入れ替わっている）
- [ ] つかみ → 相手の得 → お願い → 先に出せること → 想定問答 の骨格になっている
- [ ] 「相手の得」が自分の都合ではなく相手目線で書かれている（相互送客のWin-Winが見えている）
- [ ] お願いが「軽い一歩」から始まっている（いきなり重い協力を求めていない）

## 提携の説得力

- [ ] 「こちらが先に出せるもの（実在患者の紹介・お試し・体験会）」が具体的に入っている
- [ ] 相手の取り組みを否定していない（今のやり方を立て、隙間を補う立場になっている）
- [ ] 想定問答が、その業種の不安（③）に最低1問答えている
- [ ] 定番の反論「時間・手間が取れない」「本当に効果があるのか／怪しくないか」に答えている
- [ ] 市役所の場合、相互送客ではなく後援・共催・健康啓発のフレームになっている

## 表現のガードレール

- [ ] 「治る」「効く」と断定していない（体験・セルフケア教育として語っている）
- [ ] 医療行為・治療と誤認させる表現がない
- [ ] 医師・市役所向けでは金銭（紹介料・謝礼）を提示していない
- [ ] ガードレールを守った上で、熱量・語りの熱が死んでいない

## 話し言葉としての仕上げ

- [ ] 口調が「丁寧だが熱意のある口語」になっている（かっちりしすぎず、軽すぎず）
- [ ] 一文が長すぎない。口に出して自然に読める
- [ ] 太字（`**`）などのmarkdown装飾を本文に使っていない
- [ ] 要点が少ない場合、無理に見出しを増やして水増ししていない
- [ ] 意味の区切りで改行され、商談中に目で追いやすい
